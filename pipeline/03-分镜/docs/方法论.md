# 极客复刻 v0.9 · Round 2 Step 2 · 分镜方案 Codex 方法论（跨产品通用）

> 把任意 60s 已配音脚本转成可投放的 28-35 镜分镜表 + N 张 AI 配图 + 飞书 docx 的方法论与工具链
> 日期：2026-05-22 · 版本：v1.2 · 基础：codex-v2.1 12 模块 + 76 条极客号样本 + 3 条 MAC 微观拆解
> **适用范围**：任意 SLG / 4X 策略游戏，不限于 ROK

## v1.2 升级要点（基于 Step 3 PoC 反馈）

1. **codex 升级 v2.0 → v2.1**：新增 MAC 模块，matched_ratio ≥ 70%，sequence_group 必须「同动作多视角」而非「同主题多瞬间」
2. **镜头数升级 25 → 28-35**：贴近极客 ASL 1.6s/镜节奏中位数（原 25 镜 ASL 2.4s 偏慢 50%）
3. **storyboard 新增字段**：`sequence_group_id` / `match_type_to_prev` / `sequence_action`（主体+持续动词）
4. **product_config 新增 4 字段**：`part1_anchor` / `part2_anchor` / `part1_neg_extra` / `forbidden_language_neg`，由宿主自定义文化/服装/时代锚定
5. **资产清单升级**：新增「动作姿态 keyframe」类（每角色 3-5 张），是 Part 2 sequence_group 多视角连贯的关键
6. **首帧图设计原则**：必须画「动作进行中」姿态，给后续视频生成明确运动方向

## 一、背景与目标

Round 1 完成"脚本生成"，Round 2 Step 1 完成"口播 + BGM"。Step 2 进入"分镜方案生成"，按 codex-v2.1 11 模块约束输出可投放的分镜表 + AI 配图，是从"脚本"到"可剪辑视频"的关键中间产物。

**核心挑战**：

1. **codex 落地**：v2.0 的 11 个模块（特别是 BIN/HMI/SCT/FLU 4 个二分法宏观模块）必须显式应用，不能"模糊套用"
2. **跨产品复用**：本 pipeline 必须能服务**任意 SLG 产品**（不只是 ROK），每接新产品时机制要清晰
3. **生图风格统一**：Part 1 写实电影感 + Part 2 该产品的游戏 CG 风格，**Part 2 必须严格匹配宿主产品的官方画风**
4. **中文准确性**：AI 生图渲染中文准确率 <10%，传统做法导致 LOGO/角色名全部乱码
5. **演示完整性**：必须输出含图的飞书 docx（不只是脚本表格），让审核方能直观判断风格

## 二、5 个关键工程决策（教训驱动）

> 决策 5 是 v1.2 新增（Step 3 PoC 反馈），其他 4 个延续 v1.1


### 决策 1：换 Gemini 3 Pro Image，弃 doubao-seedream

**问题**：v1 用 doubao-seedream-4-5 跑 25 张图，**所有中文文字全部乱码**：
- LOGO："万国觉醒" → "万国凰哥" / "万坦半晖"
- 角色名："霍去病" → "霍辅"、"吕布" → "引帝"

**修正**：换 `nano_banana_pro`（Gemini 3 Pro Image），画质 + 细节 + 风格一致性更稳。

**代价**：Gemini 安全机制对"古代征服 / 多文明 / 国家旗帜"敏感会拦截大地图镜头 → fallback 到 doubao-seedream（去掉敏感词重生）。

### 决策 2：全图禁画文字，中文 100% 后期合成

**问题**：即使换 Gemini，中文准确率也不到 50%，无法做到投放级精度。

**修正**：**所有 25 张图 prompt 双重禁止文字**：
- prompt 后缀：`Clean image without any text, captions, subtitles, logo, watermark, UI text, or written language anywhere in the frame.`
- negative_prompt：`any text, any words, any letters, any numbers, captions, subtitles, logo, watermark, signage, characters, country names, written language`

**收益**：
- 中文准确性从 "<50%" 提升到 "100%（由后期保证）"
- AI 专注画画面 + 角色，不用花算力渲染文字
- 后期合成清单清晰列出每镜需加的字幕/花字/LOGO

### 决策 3：跨产品复用 — Part 2 永远用宿主官方资产做图生图

**问题**：Part 2 是游戏内画面，**每个产品的视觉风格差异极大**（ROK 卡通写实 SLG / WGAME 军事现代 / Samo 奇幻巨兽），纯文生图无法保证"该产品的真实游戏画风"。

**修正**：**接入任何新产品前的第一步永远是"跟宿主要资产"**。然后用 reference_images 图生图。

#### 通用 SOP（不限产品）

| 阶段 | 做什么 |
|---|---|
| **Step 0** | 用 `prompts/asset_request_template.md` 标准模板发给产品方/美术 owner，6 类资产（主英雄立绘 / **动作姿态 keyframe ⭐ v1.2 新增** / 兵种/建筑/LOGO/**文化锚定文字描述 ⭐ v1.2 新增**） |

| **Step 2** | 写 `product_config.yaml`（v1.2 升级 · 7 字段：基础 3 + 锚定 4），从 `prompts/product_config_template.yaml` 复制 |
| **Step 3** | 跑 `02_extract_refs.py` 拉资产 + 上传 liclick |
| **Step 4** | 跑 `03_genimg.py` 自动按 product_config 的 `part1_anchor` / `part2_anchor` / `forbidden_language_neg` 喂 prompt |

#### 各产品资产入口（项目方维护）

| 产品 | 资产 base | 主要 table |
|---|---|---|
| 你的产品 | 项目方提供 | 跟项目方/美术 owner 要 |
| 已部署 example | examples/古代将军_v3/ | 见 product_config.yaml |

> **重要**：每接入一个新产品都要登记资产 base + table 入口，沉淀团队知识。具体 ID 请向你的项目方/美术 owner 索取。

### 决策 4：SCT 主体延续（v2.0 codex 落地实操）

**v2.0 codex 规则 SCT-01**：跨 Pivot 优先 Same_Character（76 视频统计 49% 高频，比 Same_Tone 14% 强 3.5 倍）。

**实操（任意产品通用）**：
- Part 1 主体（如古代将军 / 军事专家 / 探险家）
- Pivot 27s：Hard_Cut
- Part 2 同身份角色 CG（如该产品里的对应英雄/角色立绘）

这是"现实人物 → 游戏内化身"的视觉桥梁，是极客号最高级的叙事技巧。**任何产品都适用**。

### 决策 5（v1.2 新增）：MAC + 跨产品文化锚定 + 动作姿态资产

**问题**：v1.1 跑出来的 25 镜方案，每镜都是「全新主题 + 全新主体」，相邻镜头主题完全跳跃，matched_ratio ≈ 0%，视觉碎裂。同时古代题材出现欧美面孔（模型默认偏好）+ Part 2 角色动作仅是"立绘+镜头推"。

**修正三件套**：

1. **MAC 模块（codex-v2.1）**：每段位强制 sequence_group，**「同动作多视角连贯」**（骑兵冲锋正面→战马蹄部俯视→骑兵列队侧面），不是「同主题不同瞬间」（运粮全景→冰冻骑兵→粮草地图）。判断标准：能不能用「主体+持续动词」一句话描述整组镜头？
   - ✓ 「骑兵正在持续冲锋」 → 任何机位都属于同一组
   - ❌ 「关于古代行军的杂集」 → 不是同一动作，是同一主题

2. **跨产品文化锚定**：在 `product_config.yaml` 加 `part1_anchor` / `part2_anchor` / `part1_neg_extra` / `forbidden_language_neg` 字段，由宿主自定义。示例：

| 产品 | part1_anchor | part1_neg_extra |
|---|---|---|
| ROK 古代 | "Han Dynasty Asian characters, period-accurate armor" | "non-asian people, european faces in Han Dynasty" |
| WGAME 现代军事 | "Modern NATO military equipment, realistic combat" | "medieval armor, ancient weapons, fantasy" |
| Samo 西方奇幻 | "Western medieval fantasy with dragons knights" | "modern military, real-world flags" |

3. **动作姿态资产**（v1.2 资产清单升级）：每个 P0 角色额外 3-5 张同角色多动作 keyframe。缺这类资产 → Part 2 sequence_group 必然降级为「同立绘 + 镜头推拉」的伪连贯（已实测教训）。

**生效层级**：
- Step 2（本 skill）01_gen_storyboard.py 读 part1_anchor 注入 storyboard prompt
- Step 2 03_genimg.py 跟随 storyboard 的 image_gen_prompt（已含 anchor）
- Step 3（视频生成）02_gen_video.py 读 part1_anchor 注入 video prompt + NEG

## 三、6 步 Pipeline 详解（产品无关）

### Step A · 脚本 → 分镜表（LLM · v1.2 升级）

**输入**：已配音脚本 YAML + product_config.yaml（v1.2 含 anchor 字段）
**模型**：`gemini-3.1-pro-preview`（LiteLLM Proxy）
**约束**：codex-v2.1 5 个核心文档全文（01 / 06 BIN / 07 HMI / 08 SCT / **10 MAC ⭐ v1.2 新增**）+ 产品配置
**输出**：`storyboard.json`（28-35 镜 · 含 sequence_group_id / match_type_to_prev 新字段）

```bash
python scripts/01_gen_storyboard.py <script.yaml> <product_config.yaml> <out_dir>
```

**关键 prompt 设计（v1.2）**：
1. 显式列出 codex-v2.1 12 模块的应用方式，重点 MAC：matched_ratio ≥ 70% + Part1 超大序列组 8-15 镜
2. 把 `part1_anchor` / `part2_anchor` 注入 prompt，让 LLM 写 image_gen_prompt 时按宿主定义的文化/服装/时代描述
3. 把 `forbidden_language_neg` 注入 prompt，禁用对应投放区域的禁词
4. 每镜的 `image_gen_prompt` 后缀必须含 anchor + NEG（让 03_genimg 直接复用）

### Step B · 拉宿主资产 → 上传 liclick

**输入**：`manifest.json`（按 02_extract_refs.py 的 schema 写）
**流程**：
- `source=feishu_base`：用 lark-cli api 下载 thumbnail
- `source=local`：直接复制本地图片
- 上传 liclick `upload_asset --file file_path=...` 拿 asset_id

**输出**：`asset_map.json`（角色名 → liclick asset_id）

```bash
python scripts/02_extract_refs.py <manifest.json> <out_dir>
```

### Step C · 提交 25 张图生成任务

**模型**：`nano_banana_pro`（主）+ `doubao-seedream-4-5-251128`（fallback）
**关键参数**：
- `aspect_ratio=16:9` / `image_size=2K` / `quality=high` / `n=1`
- `negative_prompt=...`（禁文字 negative 全局）
- `reference_images=[...]`（Part 2 镜头按 mapping.json 带宿主 reference）

```bash
python scripts/03_genimg.py <storyboard.json> <asset_map.json> <mapping.json> <product_config.yaml> <out_dir>
```

### Step D · Poll + 下载 + Fallback

- 10s 轮询 liclick `get_task_status`（中文匹配"完成"/"失败"）
- 完成的下载到 `images/{shot_id}.png`
- 失败的（Google 安全拦截）自动 fallback 到 doubao（去掉 prompt 敏感词）

```bash
python scripts/04_poll_download.py <task_ids.json> <storyboard.json> <out_dir>
```

### Step E · 整合 + 桌面副本

输出：`storyboard.md`（图文混排）+ 桌面副本（如指定）

```bash
python scripts/05_integrate.py <out_dir> [desktop_dir]
```

### Step F · 飞书 docx 创建 + 25 图嵌入

**关键**：lark-cli `+media-insert` 只能从 cwd 相对路径上传。

流程：`docs +create` 创建 docx → 循环每镜 `docs +update --mode append` 加 heading + 描述 → `docs +media-insert` 加图。

```bash
python scripts/06_upload_feishu.py <out_dir> "<docx 标题>"
```

## 四、codex-v2.1 11 模块在分镜中的落地（产品无关）

| 模块 | 在分镜中的体现 |
|---|---|
| STR-01 5 段结构 | Hook + Core_K + Ad_R + Sell + CTA |
| EMO-01/02/03 情绪燃料 | 每镜标 emotional_fuel（Hook 必含 Curiosity，Sell/CTA 必含 Desire_Gap） |
| PACE-01 节奏 | 镜头数按 ASL 推算 |
| SUB-01 字幕色彩 | 后期合成清单按白/金/红三色语法 |
| CAM-01/02/03 运镜 | 每镜标 camera |
| BGM-01/02 BGM 突变 | Pivot 处强反差，Selling→CTA 不突变 |
| META-01/02 隐喻 | 全片 Literal 主导，Metaphorical ≤3 |
| **BIN-01 黄金切点** ⭐v2.0 | Pivot 严格落在 [40%, 50%] |
| **BIN-02 强反差稳节奏** ⭐v2.0 | Hard_Cut + 色彩/BGM Strong 反差 |
| **BIN-03 极简硬切** ⭐v2.0 | 转场默认 Hard_Cut |
| **HMI-01/02/03 Hook 子拆** ⭐v2.0 | 疑问句首句 + 视觉锚点 + 前 3s 切 [1,3] 刀 |
| **SCT-01/02 主体延续** ⭐v2.0 | Same_Character 跨 Pivot 延续 |
| **FLU-01/02/03 Part2 流畅度** ⭐v2.0 | 画面-口播匹配 ≥85%，Build_to_Peak 情绪曲线 |
