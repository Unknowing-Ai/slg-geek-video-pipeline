# 模块 J · 主体延续性（Subject_Continuity）

## 1. 模块定位

本模块专为解决极客类“二分法”视频结构中的【Part 1（现实/科普） → Part 2（游戏展示）的跨段落缝合】问题而设。v1.0 版本侧重于单段落或单镜头内的剪辑语境（如怎么加字幕、怎么卡点），却未提供宏观视角下的断层缝合方案，导致很多视频跨越转折点（Pivot）后，观众因认知脱节而迅速流失。本模块明确规定了是否做延续、用哪种视觉线索延续，以及游戏 IP 元素的首次出场时机。

---

## 2. 核心规则

### SCT-01 同角色贯穿法则（人物一致性）
**一句话核心**：跨越转折点（Pivot）时，使用同一个人物（角色）作为认知桥梁，带观众平滑过渡到游戏世界。
**数据支撑**：在 76 条样本中，**65.8%** 的视频保留了主体延续性，而在这些做延续的视频里，高达 **48.7%** 采用了“同角色”（`Same_Character`）策略，它是维持受众认知连贯性的绝对最优解（远高于同色调或同景别）。
**执行方法**：
```json
{
  "constraint_id": "SCT-01",
  "continuity_type": "Same_Character",
  "strategy": "Character_Bridge"
}
```
*   **约束语句**：Part 1 中作为引子提及的核心人物（如历史名将、主播解说头像），必须在 Part 2 游戏画面的前 3 秒内直接作为游戏内英雄或视觉主体出场。
*   **反例**：
    *   ❌ Part 1 讲的是“诸葛亮的空城计”，Part 2 游戏画面第一眼放的是“吕布放大招”。
    *   ❌ Part 1 是真人解说，进入 Part 2 后真人解说头像直接消失，变成纯游戏画面。

### SCT-02 IP 首露时机（延迟满足）
**一句话核心**：游戏 IP 与核心 UI 的首次完整露出，必须紧跟黄金转折点（Pivot），严禁过早剧透。
**数据支撑**：游戏 IP 首次完整露出时间（`ip_reveal_s`）中位数为 **26.0s**。它严格绑定在 Pivot 刀口之后（22-26s 区间）。
**执行方法**：
```json
{
  "constraint_id": "SCT-02",
  "ip_reveal_timing": [22.0, 26.0],
  "trigger_event": "Post_Pivot"
}
```
*   **约束语句**：视频前 20 秒内（Part 1 期间）禁止出现完整的游戏界面、抽卡 UI 或明显的“游戏广告”视觉标识，需将其压抑至 22-26s 的转折点处瞬间释放。
*   **反例**：
    *   ❌ 视频开头 3 秒 Hook 阶段，就在屏幕角落挂着游戏 Logo 或下载按钮。
    *   ❌ 转折点已经过了 5 秒，画面仍在展示通用素材，未切入真实游戏 UI。

## 3. 与其他模块的协同

*   **与 BIN（二分结构）的协同**：SCT 的触发严格依赖 BIN-01 确定的转折点时间（40%-50% 进度），Pivot 刀口落下的瞬间，就是 SCT 开始接管画面视觉重心的时刻。
*   **与 CAM（运镜）的协同**：在执行 `Same_Character` 时，如果 Part 1 结尾是人物近景，Part 2 切入游戏画面时也推荐优先使用该角色的近景起幅，强化视觉对齐。
*   **与 STR（结构语法）的协同**：SCT-02 的 IP 首露通常标志着视频从 `Core_Knowledge`（或 `Background`）正式跃迁至 `Selling_Point` 或直接跳向 `Ad_Reversal`。

---

## 4. 快速 Checklist

*   ✅ **延续类型检查**：Part 1 和 Part 2 之间是否使用了相同的人物（Same_Character）作为视觉锚点？
*   ✅ **认知桥梁检查**：Part 1 的核心解说对象，是否在 Part 2 开头 3 秒内被重点展示？
*   ✅ **IP 露出检查**：全片前 20 秒是否做到“零游戏 UI/Logo 剧透”？
*   ✅ **转场时机检查**：IP 首次完整露出的时间戳是否落在 22s - 26s 区间内？
*   ❌ **反例排查**：严禁跨越 Pivot 后，角色、场景、色调同时发生毫无关联的替换。
