# 模块 K · Part 2 表达流畅度（P2_Fluency）

## 1. 模块定位

本模块聚焦解决极客类跑量素材后半段的**【Part2 内容到位】**问题。在 codex-v1.0 中，规则多侧重于单镜头内的微观操作（如字幕颜色、运镜手法），却未覆盖宏观“二分法”结构中“进入游戏展示环节后，该如何留住受众”的系统性解法。本模块通过强约束“画面-口播匹配率”、强制规划“情绪曲线（Build_to_Peak）”以及“花字命中口播率”，确保受众在跨越黄金转折点后，能够获得高度聚焦、情绪持续攀升的沉浸式“承诺兑现”体验，从而大幅降低后半段的跳出率。

---

## 2. 核心规则

### FLU-01：画面与口播极高匹配（所说即所见）
**一句话核心**：Part 2 必须严格遵循“承诺-兑现”模型，口播说到什么，画面必须立刻展示什么，拒绝任何不相关的视觉干扰。

*   **数据支撑**：在 76 条样本中，Part 2 的画面-口播匹配率（`p2_av_match_%`）中位数高达 **90.0%**。
*   **执行方法**：
    ```json
    {
      "av_match_rate": {
        "p2_av_match_min": 0.85
      }
    }
    ```
    **可读约束语句**：进入游戏展示环节后，画面内容与口播解说的匹配度必须 ≥ 85%。
*   **反例**：
    *   口播正在讲解“这把武器可以造成大范围AOE伤害”，画面却在展示主城建筑升级。
    *   使用花里胡哨的复杂转场打断了玩家对游戏核心玩法的理解（注：P2 内部应保持硬切）。

### FLU-02：情绪曲线强制 Build_to_Peak
**一句话核心**：游戏展示段落的情绪不能平淡如水，必须是持续累积直至顶点，直接引爆最后的下载指令。

*   **数据支撑**：**78.9% (60/76)** 的视频在 Part 2 阶段的情绪曲线呈现严格的“Build_to_Peak”攀升态势。
*   **执行方法**：
    ```json
    {
      "p2_emotion_flow": {
        "curve_type": "Build_to_Peak"
      }
    }
    ```
    **可读约束语句**：Part 2 情绪走向强制设为 Build_to_Peak（不断攀升），并在视频结尾推向高潮，严禁情绪中途阻断或平缓表达。
*   **反例**：
    *   高潮游戏画面放在了 Part 2 开头，随后画面越来越平淡，导致 CTA 环节情绪低迷。
    *   游戏展示段落缺乏冲突感升级，从头到尾情绪维持在同一水平线。

### FLU-03：花字高度绑定口播与核心情绪
**一句话核心**：屏幕花字绝非无意义的装饰，必须与口播词严格对齐，并精准传达特定的高转化情绪。

*   **数据支撑**：花字命中口播率（`caption_hit_%`）中位数达到 **75.0%**；且花字承载的情绪以**好奇（23.0%）**和**紧张（20.6%）**为主。
*   **执行方法**：
    ```json
    {
      "caption_sync_rate": {
        "caption_hit_min": 0.75,
        "caption_primary_emotions": ["Curiosity", "Tension"]
      }
    }
    ```
    **可读约束语句**：花字出现需与口播词高度对齐（命中率 ≥ 75%），并且核心必须承载 Curiosity（好奇）或 Tension（紧张）的情绪燃料。
*   **反例**：
    *   屏幕上自顾自弹出大量游戏术语花字，但口播根本没有提到，导致受众认知负荷过载。
    *   花字文案过于平铺直叙（如单纯写“点击升级”），没有调动玩家的紧张感或好奇心。

## 3. 与其他模块的协同

本模块（FLU）作为 v2.0 的宏观模块，需与 v1.0 的微观模块紧密配合，实现“骨架+血肉”的融合：
*   **与 PACE（节奏）协同**：FLU-01 要求 Part 2 极高的内容匹配度，这印证了 PACE 模块中“Part 2 并没有比 Part 1 剪得更快”的结论（ASL delta 中位仅 0.1s）。为了让玩家看清画面兑现的内容，切忌在 Part 2 盲目加快剪辑频率。
*   **与 EMO（情绪燃料）协同**：FLU-02 制定了 Part 2 整体必须 `Build_to_Peak` 的宏观曲线，而 EMO 模块则提供具体的微观情绪燃料（如通过具体的特效堆叠、数值暴击来体现紧张感的累积）。
*   **与 SUB（字幕颜色）协同**：FLU-03 规定了花字“什么时候出”（命中口播）以及“传达什么情绪”，而 SUB 模块的“三色语法”则规定了这些花字的视觉表现（如用金色体现转化价值，用红色体现冲击/紧张）。

## 4. 快速 checklist

*   ✅ **AV 极高匹配**：Part 2 画面展示的内容，是否与当前秒数的口播解说词做到了“所说即所见”（匹配率>85%）？
*   ❌ **无用视觉干扰**：游戏展示段落是否出现了与当前解说毫无关联的冗余操作或复杂转场？
*   ✅ **情绪持续推高**：Part 2 的游戏战斗或展示是否越来越激烈，严格遵循 Build_to_Peak 并在结尾达到最高潮？
*   ❌ **花字脱节失控**：视频中的屏幕大字是否脱离了口播节奏，变成了自说自话的“视觉光污染”？
*   ✅ **花字情绪达标**：核心弹出的花字，是否有效传递了“Curiosity（好奇）”或“Tension（紧张）”这两种高转化情绪？
