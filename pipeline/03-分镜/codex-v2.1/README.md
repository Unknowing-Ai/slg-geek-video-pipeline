# 极客视觉 Codex v2.1 — 12 模块整合规则库

> **目的**：把 v0.9.1 脚本 agent 生成的脚本，按 4 步固定流程 + 12 模块约束，变成可上线的极客风买量视频
> **方法论来源**：76 条极客号买量视频 / 382 segment（v1.0 微观）+ 同 76 条二分法 4+4 维拆解（v2.0 宏观）+ 3 条样本匹配剪辑微观拆解（v2.1 镜对镜级别）三重归纳
> **适用产品**：ROK 万国觉醒 / 战火勋章 / 万龙觉醒 等莉莉丝 SLG 全产品
> **v2.1 升级要点**：在 v2.0 11 模块基础上新增 MAC（匹配剪辑）模块，解决"硬切 71% 但看着不突兀"的真正机制 — 段位内部用「同主题多素材混剪」(81% 占比) 让相邻硬切镜头接住，这是 Step 3 视频生成 PoC 暴露的最大 codex gap

---

## 整体流水线（与 v1.0 一致）

```
[v0.9.1 脚本输出]
       ↓
①  脚本 → 分镜表  ──→  分镜表（每镜：时间/画面/运镜/素材类型）
       ↓
②  分镜表 → 视频片段  ──→  N 个 mp4 片段（每个 1.5-3s）
       ↓
③  口播 → 字幕 + 音轨  ──→  字幕样式稿 + BGM 选型 + SFX 卡点表
       ↓
④  片段 → 成片（剪辑+转场）  ──→  完整 60s 成片 mp4
```

## 11 大底层规则模块（贯穿 4 步）

### v1.0 微观模块（段位/镜头级 · 7 模块）

| 模块 | 涉及哪步 | 核心约束 |
|---|---|---|
| **A 结构语法 STR** | ① | 默认 5 段（Hook→Core_Knowledge→Ad_Reversal→Selling_Point→CTA），上限 5 段，下限 3 段，Background 单视频最多 1 次 |
| **B 情绪燃料 EMO** | ①③ | Hook 必含 Curiosity / Selling+CTA 必含 Desire_Gap / 民族优越感单视频上限 1 次 |
| **C 节奏 PACE** | ①④ | 全视频 ASL 目标 2.0-2.5s，单段禁止连续 4+ 个 ASL<1.0s 镜头 |
| **D 字幕色彩 SUB** | ③ | 白=信息 / 金=价值 / 红=冲击；单视频颜色种类上限 3 种 |
| **E 运镜 CAM** | ①② | Hook=Mixed+Zoom_In / CTA=Zoom_Out/Static，禁密集 Zoom_In / Core=Mixed+Pan |
| **F BGM 突变 BGM** | ③④ | Ad_Reversal 起点强制突变 / Selling→CTA 不突变（情绪延续） |
| **G 隐喻密度 META** | ②③ | Literal 主导（88.7%），Metaphorical 限 Hook+Core_Knowledge，单视频上限 3 个 |

### v2.0 新增宏观模块（结构/缝合级 · 4 模块）

| 模块 | 涉及哪步 | 核心约束 | 详细文档 |
|---|---|---|---|
| **H 二分结构 BIN** | ①④ | Pivot 落在 40-50%（中位 46%）/ 强反差 Strong / Hard_Cut 主导 / Part1/Part2 ASL Δ ≤ 0.2s | [06-二分结构-BIN.md](06-二分结构-BIN.md) |
| **I Hook 0-3s 子拆 HMI** | ①② | Question 主导抛出方式（53%）/ 前 3s 视觉锚点命中率 ≥80% / 前 3s 切刀 [1-3] | [07-Hook-0-3s-HMI.md](07-Hook-0-3s-HMI.md) |
| **J 主体延续性 SCT** | ②④ | 跨 Pivot 优先 Same_Character（49% 高频）/ IP 首露紧跟 Pivot（中位 26s） | [08-主体延续性-SCT.md](08-主体延续性-SCT.md) |
| **K Part 2 流畅度 FLU** | ②③④ | 画面-口播匹配率 ≥85% / 情绪曲线 Build_to_Peak / 花字命中口播率 ≥75% | [09-Part2流畅度-FLU.md](09-Part2流畅度-FLU.md) |

### v2.1 新增镜对镜级模块（剪辑衔接级 · 1 模块）

| 模块 | 涉及哪步 | 核心约束 | 详细文档 |
|---|---|---|---|
| **L 匹配剪辑 MAC** | ①②④ | matched_ratio ≥ 70% / 主力 type 为 Same_Theme_Multi_Source（81%）/ 最大序列组 ≥ 8 镜 / Part1 单超大组 8-15 镜 | [10-匹配剪辑-MAC.md](10-匹配剪辑-MAC.md) |

## 反 baseline 强信号（颠覆直觉 · v1.0 + v2.0 + v2.1 合集）

| 旧认知 | 数据反驳 | 真实结论 | 来源 |
|---|---|---|---|
| "段位越完整越好" | 高 CTR 平均 4.88 段，低 CTR 5.54 段 | **少而精 > 多而碎** | v1.0 |
| "Background 段必须有" | 高 CTR 21% 含 Background，低 CTR 46% 含 | **Background 段拖累 CTR** | v1.0 |
| "短视频要密集快切" | 高 CTR ASL 2.22s，低 CTR 1.75s | **慢镜头反而打** | v1.0 |
| "民族优越感是必杀技" | 高 CTR 0.67 次/视频，低 CTR 0.83 次/视频 | **民族优越感被滥用** | v1.0 |
| "Part 2 要快切提速" | Part1/Part2 ASL Δ 中位仅 0.1s（76 条样本） | **节奏均一，不靠快切提速** | v2.0 ⭐ |
| "转场要有酷炫特效音" | 39.5% 的 Pivot 转场音效为 None / 71% 用纯 Hard_Cut | **极简硬切 > 花式转场** | v2.0 ⭐ |
| "场景延续靠色调氛围" | 66% 视频做主体延续，其中 49% Same_Character / 仅 14% Same_Tone | **跟着人物进游戏 > 色调铺垫** | v2.0 ⭐ |
| "硬切 71% 必然导致碎片化" | 段内 81% 切刀是匹配剪辑（同主题/同主体），matched_ratio 中位 82% | **硬切+匹配剪辑 = 极客号"看着不断"的真机制** | v2.1 ⭐ |
| "同场景多视角才能消除硬切感" | Same_Scene_Multi_Angle 占比 0%（极客号根本不拍原片） | **影视混剪靠的是"同主题不同素材"，不是"同场景不同机位"** | v2.1 ⭐ |

## 怎么用这套 codex

**新手路径**：
1. 先读 [05-完整示例-百万大军.md](05-完整示例-百万大军.md)（来自 v1.0），看一条视频从脚本到成片的完整过程
2. 再读 [06-二分结构-BIN.md](06-二分结构-BIN.md)，理解 v2.0 的宏观骨架
3. 对照自己脚本应用 01-09 文档

**老手路径**：
1. 直接看 [01-脚本到分镜.md](01-脚本到分镜.md) 的"分镜表模板"和"自动转换 prompt"
2. 用 v1.0 7 模块出微观规则约束 + v2.0 4 模块加宏观约束，组合喂 LLM
3. 按 02-04 工具/命令模板执行

**团队协作路径**：
- 文案：按 v0.9.1 codex 出脚本
- 分镜师：按 01 + 06（BIN 二分位置）+ 07（HMI Hook 子拆）出分镜表
- 视频制作：按 02 + 08（SCT 主体延续）找素材
- 剪辑师：按 03 + 04 + 09（FLU 流畅度）完成字幕音轨剪辑
- 风格审核官（Step 4）：按 11 模块 checklist 逐条验收，不通过回 Step 2
