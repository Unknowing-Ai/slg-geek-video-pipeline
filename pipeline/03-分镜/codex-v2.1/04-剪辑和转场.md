# Step 4: 视频片段 → 成片（剪辑 + 转场）

> **输入**：所有镜头 mp4 片段 + ASS 字幕脚本 + BGM mp3 + SFX wav 列表
> **输出**：完整 60s 成片 mp4（含字幕、BGM、SFX、调色）
> **谁做**：FFmpeg 自动化 / 剪辑师（Premiere/达芬奇）人工
> **关键约束**：段位转场必须遵守突变规则 / 三轨音频精确同步 / 调色按段位渐变

---

## A. 5 步剪辑流程

```
1. 镜头拼接（按分镜表时间序列硬切）
       ↓
2. 段位转场处理（Ad_Reversal 加黑屏+特效）
       ↓
3. 字幕叠加（ASS 文件烧录）
       ↓
4. 音轨混合（口播+BGM+SFX 三轨对齐）
       ↓
5. 整体调色（按段位渐变色调）
```

---

## B. 转场规则

### B1. 段位内转场（硬切为主）

| 转场方式 | 适用 | 占比 | 注意 |
|---|---|---|---|
| **硬切**（cut）| 段位内 90% 镜头 | 主导 | 默认选择，最干净 |
| **匹配剪辑**（match cut）| 动作 / 形状 / 色彩匹配 | <5% | 高级技巧，少用 |
| **黑屏淡入淡出** | 同段内禁用 | 0% | 只用于段位间 |
| **动效转场**（推拉/旋转/故障）| 段位内禁用 | 0% | 只用于 Ad_Reversal |

### B2. 段位间转场（按规则）

| 段位转场 | 操作 | FFmpeg 实现 |
|---|---|---|
| Hook → Core_K | 硬切 | 直接拼接 |
| Core_K → Ad_Reversal | **黑屏 0.3s + Glitch 转场 + BGM 戛然** | 见 D2 |
| Background → Ad_Reversal | **黑屏 0.3s + Whoosh 音效 + BGM 突变** | 见 D2 |
| Ad_Reversal → Selling_Point | 硬切（BGM 已切到史诗音乐） | 直接拼接 |
| Selling_Point → CTA | 硬切（**BGM 必须延续不变**） | 直接拼接 |

---

## C. 调色规则

按段位的"视觉情绪曲线"做调色：

| 段位 | 调色方向 | 饱和度 | 对比度 | 色温 |
|---|---|---|---|---|
| Hook | 暗色铺垫，悬疑感 | -10% | +5% | 偏冷（-200K） |
| Background | 中性 | 0% | 0% | 中性 |
| Core_Knowledge | 偏暗（知识科普氛围）| -5% | 0% | 中性 |
| **Ad_Reversal** | **突亮（视觉冲击）** | **+20%** | **+15%** | **偏暖（+300K）** |
| Selling_Point | 高饱和（卖点鲜艳）| +15% | +10% | 偏暖（+200K） |
| CTA | 收束（行动指令突出）| +10% | +20% | 偏暖（+100K） |

**整体曲线**：暗 → 暗 → 暗 → **突亮** → 亮 → 亮，对应"知识沉浸 → 商业爆发"

## D. FFmpeg 自动化命令

### D1. 镜头拼接（concat demuxer）

先做一个 `concat_list.txt`：
```
file 'shots/shot_01_hook_1.mp4'
file 'shots/shot_02_hook_2.mp4'
file 'shots/shot_03_core_1.mp4'
...
```

然后：
```bash
ffmpeg -f concat -safe 0 -i concat_list.txt -c copy concat.mp4
```

### D2. Ad_Reversal 段位转场（黑屏 + 音效）

```bash
# 在 Core_K 最后一个镜头和 Ad_R 第一个镜头之间插入黑屏 0.3s
ffmpeg -f lavfi -i color=black:size=1080x1920:duration=0.3 \
       -f lavfi -i anullsrc=channel_layout=stereo:sample_rate=44100 \
       -t 0.3 -c:v libx264 -pix_fmt yuv420p -c:a aac black_03s.mp4

# 然后用 concat 拼接：core_last + black_03s + ad_first
ffmpeg -f concat -safe 0 -i transition_list.txt -c copy with_transition.mp4
```

### D3. 字幕叠加（烧录 ASS）

```bash
ffmpeg -i with_transition.mp4 -vf "ass=subtitle.ass" -c:a copy with_subtitle.mp4
```

### D4. 音轨混合（口播 + BGM + SFX）

3 轨混音：
```bash
ffmpeg -i with_subtitle.mp4 \
       -i voiceover.wav \
       -i bgm.mp3 \
       -i sfx_track.wav \
       -filter_complex "[1:a]volume=1.0[voice]; \
                        [2:a]volume=0.15[bgm]; \
                        [3:a]volume=0.6[sfx]; \
                        [voice][bgm][sfx]amix=inputs=3:duration=longest[aout]" \
       -map 0:v -map "[aout]" -c:v copy -c:a aac final_mix.mp4
```

**音量参数**：
- voiceover: 1.0（最响）
- bgm: 0.15（低背景）
- sfx: 0.6（中等突出）

### D5. SFX 精确卡点（用 amerge + adelay）

每个 SFX 单独 delay 到对应时间点：
```bash
ffmpeg -i sfx_boom.wav -i sfx_ding.wav -i sfx_whoosh.wav \
       -filter_complex "[0:a]adelay=2500|2500[boom1]; \
                        [1:a]adelay=18000|18000[ding1]; \
                        [2:a]adelay=25000|25000[whoosh1]; \
                        [boom1][ding1][whoosh1]amix=inputs=3:duration=longest[sfx_combined]" \
       sfx_track.wav
```

`adelay=2500|2500` 意思：左右声道都延迟 2500ms（精确到毫秒）

### D6. 调色（按段位渐变 LUT）

每个段位用一个 LUT 或者 curves filter：
```bash
ffmpeg -i input.mp4 -vf "curves=preset=darker:all='0/0 0.5/0.4 1/0.95',\
                         eq=saturation=0.9:contrast=1.05" \
       -t 25 hook_to_core_graded.mp4
```

## E. 自动化 vs 人工的边界

| 操作 | 自动化可行性 | 推荐做法 |
|---|---|---|
| 镜头拼接 | ✅ 完全自动 | FFmpeg concat |
| 段位转场（黑屏） | ✅ 完全自动 | FFmpeg 模板 |
| 字幕叠加 | ✅ 完全自动 | ASS 文件 + FFmpeg ass filter |
| 音轨混合（口播+BGM 主轨） | ✅ 完全自动 | FFmpeg amix |
| SFX 卡点 | ✅ 完全自动 | FFmpeg adelay（前提是 SFX 卡点表完整） |
| 整体调色（段位渐变） | 🟡 半自动 | FFmpeg curves + 人工微调 |
| 转场决策（什么时候用匹配剪辑）| ❌ 人工 | 剪辑师判断 |
| 镜头时长微调（节奏感）| ❌ 人工 | 剪辑师听感把握 |

**结论**：80% 的剪辑工作可以 FFmpeg 全自动，20% 需要剪辑师 review 和微调。

## F. 端到端 Bash 脚本模板

```bash
#!/bin/bash
# 极客视觉 codex v1.0 - 自动剪辑 pipeline
# Usage: ./build_video.sh <project_dir>

PROJECT=$1
cd "$PROJECT"

# 1. 镜头拼接（按分镜表顺序）
ffmpeg -f concat -safe 0 -i concat_list.txt -c copy step1_concat.mp4

# 2. 段位转场处理（插入黑屏）
ffmpeg -f concat -safe 0 -i transition_list.txt -c copy step2_transition.mp4

# 3. 字幕叠加
ffmpeg -i step2_transition.mp4 -vf "ass=subtitle.ass" -c:a copy step3_subtitle.mp4

# 4. SFX 卡点合成
ffmpeg [...SFX adelay 混合命令...] sfx_track.wav

# 5. 音轨混合
ffmpeg -i step3_subtitle.mp4 -i voiceover.wav -i bgm.mp3 -i sfx_track.wav \
       -filter_complex "[1:a]volume=1.0[voice]; \
                        [2:a]volume=0.15[bgm]; \
                        [3:a]volume=0.6[sfx]; \
                        [voice][bgm][sfx]amix=inputs=3:duration=longest[aout]" \
       -map 0:v -map "[aout]" -c:v copy -c:a aac step4_mixed.mp4

# 6. 整体调色（按段位时间段）
ffmpeg -i step4_mixed.mp4 -filter_complex "[...curves filter...]" final.mp4

echo "Done: final.mp4"
```

## G. 输出参数（投放格式）

成品 mp4 参数：

| 参数 | 数值 | 说明 |
|---|---|---|
| 分辨率 | 1080×1920 | 抖音/TikTok 标准竖屏 |
| 帧率 | 30fps | 流畅且文件不大 |
| 视频编码 | H.264 | 兼容性最好 |
| 视频码率 | 4-6 Mbps | 投放质量 |
| 音频编码 | AAC | 标准 |
| 音频码率 | 192 kbps | 立体声 |
| 文件大小目标 | ≤ 50 MB | 投放上传顺畅 |

最终输出命令：
```bash
ffmpeg -i final.mp4 -c:v libx264 -preset slow -crf 23 \
       -c:a aac -b:a 192k -movflags +faststart \
       output_for_投放.mp4
```
