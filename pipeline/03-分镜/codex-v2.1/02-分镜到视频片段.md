# Step 2: 分镜表 → 视频片段

> **输入**：单个镜头的完整描述（时间 / 画面 / 运镜 / 素材类型）
> **输出**：实际的 mp4 视频片段（每个 1.5-3s）
> **谁做**：kling + doubao-seedance 视频生成（自动生成）/ 素材库（人工筛选）/ AE Canva（动态图表）/ 真人补拍
> **关键约束**：Literal（具象）镜头占 88.7%，避免过度依赖 AI 生成抽象画面

---

## 素材来源决策树

每个镜头先判断"画面是什么"，然后走对应路径：

```
镜头需要什么素材？
├─ 历史/影视场景（如古代战场、官员奏报）
│  └─ Movie_Footage → 历史素材库 / B-roll 网站
│     ├─ 资源：getty / pond5 / 莉莉丝内部历史素材库
│     └─ 注意：版权 risk，商用必须授权
│
├─ 游戏内画面（如 ROK 战场、名将、UI）
│  └─ Game_CG → 游戏内录屏 + 截图
│     ├─ 工具：OBS / 游戏内置截图
│     └─ 注意：建议用最新版本游戏画质
│
├─ 数据可视化 / 动态图表
│  └─ Motion_Graphics → AE / Canva / kling + doubao-seedance 视频生成
│     ├─ 简单（数字 Pop_Up）：AE 预设模板
│     ├─ 中等（流程图/对比图）：Canva 模板
│     └─ 复杂（创意动画）：kling + doubao-seedance 视频生成 生成
│
├─ 真人场景（如出镜讲解、生活画面）
│  └─ Live_Action → 素材库 / 真人补拍
│     ├─ 优先：现有出镜素材库
│     └─ 没有：列入补拍清单（成本高）
│
├─ 抽象/隐喻画面（如"情绪""时间"）
│  └─ kling + doubao-seedance 视频生成 生成 / Motion_Graphics
│     └─ 注意：META-02 上限 3 个/视频
│
└─ 静态图片（如人物立绘、标志图）
   └─ Static_Image → kling + doubao-seedance 视频生成 文生图 / 素材库
```

## kling + doubao-seedance 视频生成 适用范围

CLAUDE.md 已配置kling + doubao-seedance 视频生成（liclick），调用方式：
```bash
atlas-skillhub gateway call-tool --service liclick --tool generate_image --args '{...}'
atlas-skillhub gateway call-tool --service liclick --tool generate_video --args '{...}'
```

### ✅ 推荐用kling + doubao-seedance 视频生成 的场景

| 场景 | 优势 | 注意 |
|---|---|---|
| **Motion_Graphics 动态图表** | 单镜头短（1.5-2.5s），无角色一致性需求 | prompt 写清楚色调、构图、动画方向 |
| **隐喻画面**（如"崩溃→城墙倒塌"） | 抽象概念用 AI 比找实拍更快 | 单视频 ≤3 个 |
| **Static_Image 静态图**（如开场标题板、名将立绘）| 文生图最快 | 用 generate_image 而非 video |
| **过渡转场镜头**（如黑屏 + 光效） | 短而简单 | 时长建议 ≤2s |

### ❌ 不推荐用kling + doubao-seedance 视频生成 的场景

| 场景 | 为什么不行 |
|---|---|
| **复杂运镜**（Mixed + 多次 Zoom） | 莉刻镜头连贯性受限，多次运镜易穿帮 |
| **含特定游戏 IP**（ROK 万国觉醒名将、UI、地图）| 莉刻不知道 ROK，必须用游戏内录屏 |
| **真实历史人物**（如成吉思汗、白起）| 容易生成不像/争议画面，建议用影视素材 |
| **长镜头**（>3s） | 莉刻视频生成单次时长限制 + 画面易扭曲 |
| **多角色互动场景** | 角色一致性差，多人画面易混乱 |

## kling + doubao-seedance 视频生成 Prompt 模板

每个镜头的莉刻 prompt 包含 5 个要素：

```
[场景] + [运镜] + [视觉风格] + [画面调性] + [时长]
```

### 模板 1: Motion_Graphics 动态图表

```json
{
  "prompt": "数据可视化动画：屏幕中央显示数字'100万'，慢慢分解成'作战5万'和'精锐3万'两个小数字，
            动画时长2秒，黑底白字风格，关键数字带金色高亮 #FFD700，
            镜头静止 Static，整体调性严肃科普",
  "duration_seconds": 2,
  "aspect_ratio": "9:16",
  "style": "infographic"
}
```

### 模板 2: 隐喻画面

```json
{
  "prompt": "古代民夫推粮车，雪山雨夜泥泞道路，画面色调灰暗压抑，
            镜头缓慢横移 Pan，2.5秒，电影质感，体现Tension紧张氛围",
  "duration_seconds": 2.5,
  "aspect_ratio": "9:16",
  "style": "cinematic_dark"
}
```

### 模板 3: 静态图片（用 generate_image）

```json
{
  "prompt": "极客风视频开场标题板：'百万大军，其实只有几千？'，
            黑底白字大标题居中，'百万'用红色 #FF0000 高亮，'几千'用白色,
            字体粗黑体，9:16 竖屏",
  "aspect_ratio": "9:16"
}
```

### 复用率统计（重要）

每条新视频做完，记录"哪些素材是新生成的 vs 复用的"：
- 高复用率（>60% 素材复用）= pipeline 效率高
- 低复用率（<30% 复用）= 每条都要新做，成本高，要审视

## 成本与时间估算

| 素材类型 | 单镜头成本 | 单镜头时间 | 备注 |
|---|---|---|---|
| Movie_Footage 复用 | ¥0 | 5 分钟（找+剪）| 前提是素材库已建好 |
| Movie_Footage 新购 | ¥50-500 | 1-2 天 | 走 getty 等 |
| Game_CG 录屏 | ¥0 | 10-30 分钟 | 需要进游戏录 |
| Motion_Graphics AE | ¥0 | 30 分钟 | 用模板套 |
| Motion_Graphics 莉刻 | ¥1-3 | 1-3 分钟 | 单镜头生成 |
| Live_Action 复用 | ¥0 | 5 分钟 | 素材库已有 |
| Live_Action 补拍 | ¥500-2000 | 2-5 天 | 需要约人拍摄 |
| Static_Image 莉刻 | ¥0.5-1 | 30 秒 | 文生图 |

### 一条 60s 视频总成本估算

假设 25 个镜头 / 素材复用率 70%：
- 17 个镜头复用 ¥0
- 5 个镜头 Motion_Graphics（3 个 AE + 2 个莉刻）¥6
- 3 个镜头 Game_CG 录屏 ¥0
- **总素材成本 ~¥6**（不含人工时间）

如果完全从零做（复用率 0%）：
- 25 个镜头 × 平均 ¥30 = **¥750**（含部分莉刻 + 部分新购素材）

**结论**：素材库越完善，单条成本越低。建议把每条做完的成片素材都归档进 `materials/`，半年后复用率应能达到 80%+。
