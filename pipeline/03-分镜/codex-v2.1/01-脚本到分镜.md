# Step 1: 脚本 → 分镜表

> **输入**：v0.9.1 脚本 agent 生成的脚本（含口播 + 字幕 + 段位结构）
> **输出**：分镜表（每镜：时间区间 / 画面描述 / 运镜 / 素材类型 / 情绪燃料）
> **谁做**：Claude / Gemini 自动转换（喂 prompt 即可），或人工

---

## 转换逻辑（7 步）

拿到一段脚本，依次走 7 个判断：

### 1. 识别段位

按口播内容判定每段属于哪个 `semantic_role`（v0.9.1 已经标过，这步可跳过）：

| 段位 | 识别特征 |
|---|---|
| Hook_Counter_Intuitive | 视频开头，疑问句或反共识陈述 |
| Background | 紧跟 Hook，做世界观/前提铺垫（**可选，能不要就不要**） |
| Core_Knowledge | 解答 Hook 的核心知识 |
| Ad_Reversal | 从知识切到游戏的过渡（关键反转） |
| Selling_Point | 游戏卖点轰炸 |
| CTA | 结尾行动指令 |

### 2. 决定段位时长

按 76 视频数据的平均时长（必要时按口播字数微调）：

| 段位 | 平均时长 | 时长占比 | 建议范围 |
|---|---|---|---|
| Hook | 7.8s | 15.8% | 3-10s |
| Background | 10.3s | 8.7% | 5-12s（可选）|
| Core_Knowledge | 12.0s | 21.8% | 8-20s |
| Ad_Reversal | 9.4s | 15.1% | 5-15s |
| Selling_Point | 11.0s | 27.7% | 8-15s |
| CTA | 5.5s | 10.8% | 3-8s |
| **全片合计** | **46.5s** | 100% | 45-60s |

### 3. 决定镜头数量（ASL 决定）

按 PACE-01 规则，全视频 ASL 目标 2.0-2.5s：

| 段位 | 推荐 ASL | 镜头数 = 段位时长 / ASL |
|---|---|---|
| Hook | 1.5-2.0s（最快） | 4-6 个 |
| Background | 2.5s | 4-5 个 |
| Core_Knowledge | 2.0-2.2s | 5-10 个 |
| Ad_Reversal | 2.0s | 5-8 个 |
| Selling_Point | 2.5s | 4-7 个 |
| CTA | 3.0s（最慢）| 2-3 个 |

**红线**：单段位禁止连续 4+ 个 ASL<1.0s 的镜头（破坏认知节奏）

### 4. 决定每段运镜（CAM-01/02/03）

| 段位 | 运镜 | 理由 |
|---|---|---|
| Hook | Mixed（必含 ≥1 个 Zoom_In） | 制造冲击 |
| Background | Pan / Static | 平铺铺垫 |
| Core_Knowledge | Mixed + Pan | 横向滑过数据/列表 |
| Ad_Reversal | Zoom_In → Zoom_Out | 聚焦反转点再展开 |
| Selling_Point | Mixed（Zoom_In 名将/卖点 + Pan 战场） | 多元展示 |
| **CTA** | **Zoom_Out / Static**（**禁密集 Zoom_In**） | 给观众反应时间 |

### 5. 决定素材类型（asset_type_ratio）

按 76 视频的实测分布：

| 段位 | 主要素材类型 |
|---|---|
| Hook | Movie_Footage 80%（历史场景 / 真实画面）+ Live_Action 20% |
| Background | Movie_Footage 70% + Motion_Graphics 30% |
| Core_Knowledge | Movie_Footage 60% + Motion_Graphics 30%（数据可视化） + Static_Image 10% |
| **Ad_Reversal** | **Game_CG 60-80%（必切换到游戏素材）** + Movie_Footage 20-40% |
| Selling_Point | **Game_CG 80-100%**（游戏内画面为主） |
| CTA | Game_CG 100%（LOGO + 战场背景） |

**关键转折**：Ad_Reversal 起点是素材类型从 Movie/Live → Game_CG 的硬切换点（与 BGM 突变同步）

### 6. 决定每段情绪燃料（EMO-01/02/03）

| 段位 | 主燃料 | 副燃料（可选） |
|---|---|---|
| Hook | **Curiosity（必含，≥50%）** | Tension |
| Background | Intellectual_Superiority / Nostalgia | None |
| Core_Knowledge | **任一种 ≤40%**（保持均衡）| 必含 Intellectual_Superiority |
| Ad_Reversal | **Desire_Gap（必含）** | — |
| Selling_Point | **Desire_Gap** | Ethnic_Superiority（限 1 次/全视频） |
| CTA | **Desire_Gap（延续 Selling）** | — |

**民族优越感（Ethnic_Superiority）规则**：全视频出现上限 1 次，推荐放在 Core_Knowledge 或 Selling_Point 段

### 7. 决定隐喻 vs 具象（META-01/02）

| 段位 | 推荐类型 |
|---|---|
| Hook | Literal（前 3 秒不能让人猜，需直接看懂） |
| Background | Literal |
| Core_Knowledge | Literal 为主，可插 1-2 个 Metaphorical（如"缺粮"对应"民夫推车泥泞"）|
| Ad_Reversal | Literal（游戏画面真实展示）|
| Selling_Point | Literal（卖点必须看得清）|
| CTA | Literal |

**红线**：全视频 Metaphorical 镜头 ≤ 3 个（过度隐喻增加解码成本）

## 自动转换 Prompt（喂给 Claude/Gemini）

如果你想让 AI 自动从脚本生成分镜表，用下面这段 prompt：

```
你是极客视觉 codex v1.0 的分镜师。给你一段 v0.9.1 生成的极客风脚本，
你需要按 codex 规则把它转成分镜表。

# 脚本输入
[在此粘贴 v0.9.1 脚本，含段位标记 + 口播 + 字幕]

# 转换规则（必须严格遵守）

## 段位时长
Hook 3-10s / Background 5-12s（可选）/ Core_Knowledge 8-20s / 
Ad_Reversal 5-15s / Selling_Point 8-15s / CTA 3-8s
全片 45-60s

## 镜头节奏（ASL 单镜头时长）
Hook 1.5-2.0s / Background 2.5s / Core_Knowledge 2.0-2.2s / 
Ad_Reversal 2.0s / Selling_Point 2.5s / CTA 3.0s
红线：单段禁止连续 4+ 个 ASL<1.0s 镜头

## 运镜
Hook: Mixed (必含 Zoom_In) / Background: Pan/Static / 
Core_Knowledge: Mixed+Pan / Ad_Reversal: Zoom_In→Zoom_Out /
Selling_Point: Mixed / CTA: Zoom_Out 或 Static（禁密集 Zoom_In）

## 素材类型（asset_type_ratio）
Hook: Movie_Footage 主 / Background: Movie+Motion_Graphics /
Core_Knowledge: Movie+Motion_Graphics / 
Ad_Reversal: 从 Movie 切到 Game_CG（必须切换）/
Selling_Point: Game_CG 主 / CTA: Game_CG 主

## 情绪燃料
Hook: Curiosity 必含（≥50%）/ Background: Intellectual_Sup 或 Nostalgia /
Core_Knowledge: 均衡（任一类 ≤40%）必含 Intellectual_Sup /
Ad_Reversal: Desire_Gap 必含 / Selling_Point: Desire_Gap +可选 Ethnic_Sup（全视频 1 次）/
CTA: Desire_Gap 延续

## 隐喻
全视频 Metaphorical 镜头 ≤3 个，限 Hook+Core_Knowledge 使用

# 输出要求

输出 markdown 表格：
| 段位 | 镜头# | 时间区间 | 时长 | 画面描述 | 运镜 | 素材类型 | 情绪燃料 | 备注 |

每个画面描述要具体（人物/动作/构图/对象），不能模糊。
不要凭空捏造脚本中没有的内容。
```
