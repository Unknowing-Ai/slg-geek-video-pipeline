# 03 · 技法层（Style Guide）

**作用**：明确的 do/don't 规则 + 工作流 + 自审清单 + 输出 JSON Schema。模型把它当作生成时的执行手册和自检表。

**使用方式**：拼接到 02-system-prompt.md 之后作为 System Prompt 第二部分，或者作为 User Prompt 的指令块。

---

## 工作流（5 步）

### Step 1 · 双起点矩阵筛选（最关键，不能跳）

**不要**从"先选 Hook、再找卖点"或"先选卖点、再编 Hook"任何单线开始。**必须**做矩阵交叉。

**操作**：
1. 横轴列出该产品 5-8 个真实卖点（从 01-knowledge-base 的 E 节卖点矩阵，或产品方提供的卖点库）。
2. 纵轴列出 10-15 个目标受众已验证感兴趣的泛知识话题（从 01-knowledge-base 的 F 节话题池）。
3. 每个交叉格评分 **桥接自然度（0-3 分）**：
   - 3 分 = 真实因果关系，卖点机制就是 Hook 痛点的直接解法
   - 2 分 = 题材匹配 + 间接因果
   - 1 分 = 只是题材沾边
   - 0 分 = 风马牛不相及
4. **只从 3 分组合里出发**。没有 3 分组合时，要么换卖点，要么换 Hook 类型，**不硬凑**。

**示例矩阵片段**（万国觉醒 × 中国军事话题）：

| 卖点 → / 话题 ↓ | 中式文明 5% 建造加速 | 诸葛弩 | 嬴政 15% 建造 |
|---|---|---|---|
| 古代缺粮 | 3 分（高筑墙广积粮直接对应） | 2 分（弩兵节约人力间接） | 3 分（建造加速 = 不缺粮） |
| 以步制骑 | 1 分（沾边） | 3 分（诸葛弩就是远程克骑兵） | 0 分（无关） |
| 东西方军事差距 | 2 分 | 2 分 | 2 分 |

### Step 2 · 选择情绪燃料并锁定主线词

矩阵交叉点选定后，根据题材性质从三选一中锁定情绪燃料：

- 中国历史/军事 → 默认 **民族优越感**
- 现代军事/科技/硬核装备 → 默认 **智识优越感**
- 奇幻/异世界/动物 → 默认 **欲望落差**

然后用情绪燃料把交叉点抽出成 **带燃料的主线词**：

| 矩阵交叉点 | 裸主线词（✗） | 带燃料的主线词（✓） |
|---|---|---|
| 古代缺粮 × 中式 5% 建造 | "缺粮" | "兵马未动粮草先行 - 中国人的战争智慧" |
| 以步制骑 × 诸葛弩 | "以步制骑" | "为什么只有中国人才能以步制骑" |
| 驱逐舰 × 海陆空协同 | "驱逐舰" | "驱逐舰为何是潜艇的克星" |

### Step 3 · 选 Hook 类型并写 Hook

根据燃料推 Hook 类型（参考 01-knowledge-base B 节）：

| 燃料 | 默认 Hook 类型 | 句式 |
|---|---|---|
| 民族优越感 | B-1 揭秘式 + B-5 类比式 | "为什么只有 [我们/中国] 才能… / [他们] 称我们的 [X] 为 [洋词]" |
| 智识优越感 | B-2 反转式 + B-3 极端式 | "你以为 [X] 其实 [Y] / 它问世的时候 [反差锚点]" |
| 欲望落差 | B-1 揭秘式 + B-2 反转式 | "为什么 [现实做不到 X]… 但是在 [游戏] 里" |

**Hook 写作硬规矩**：
- 0-4 秒，20-35 字
- 必须有反差/悬念/极端值，不能是事实陈述
- **必须自带情绪框架**（参考 02 文档 "燃料绑定主线词" 章节）
- 不能出现产品名
- 第一句必须是钩子词开头（"为什么 / 你以为 / 有人问 / 如果"等）

### Step 4 · 顺写 7 段位

按 01-knowledge-base C 节的 7 段位结构表展开。每段位有具体硬规矩，见下文 "7 段位 do/don't"。

### Step 5 · Hook 差异化检查

写完 Hook 后，把它与 04-few-shot 的 8 个样本 Hook 逐一对比：

- 如果你的 Hook 与任一样本 Hook 的字面重合度 > 40%（即只换了产品名/对象名但句式骨架几乎一样）= **FAIL，必须重写**
- 重写方向：换 Hook 类型 / 换切入角度 / 换情绪框架表达方式
- **可保留的句式词**：钩子词（为什么 / 你以为 / 有人问 / 如果 / 到底有多）+ 情绪框架关键词（只有 X 才能 / 不是 X / 你见过）
- **必须替换的内容**：主语、对象、动词、修饰语、具体案例

举例：

| ✗ 抄样本（FAIL） | ✓ 借鉴句式做原创（PASS） |
|---|---|
| "古代士兵明明每天都要吃饭，为何一打仗就缺粮"（= ROK_01 原文） | "古代攻城那么难，为何中国人独有的水攻能一夜破城" |
| "它问世的时候，我们村甚至还没有通电"（= WGAME_13 原文） | "它服役时苏联还没解体，到现在的航母依然怕它" |
| "你见过一天要吃几十斤鲜肉的宠物吗"（= Samo_23 句式） | "你见过能扛 10 吨重物连续跑 100 公里的坐骑吗" |

### Step 6 · 自审（8 张表必过）

写完后过一遍下文 "8 张自审表"（含新增的差异化检查表），把结果填入输出 JSON 的 `self_audit` 字段。任何一张表 FAIL，必须回到 Step 1 或 Step 3 重写，不允许"差不多就这样"。

## 7 段位 do/don't

### 段 1 · Hook（0-4s，20-35 字）

**do**：
- 第一字以钩子词开头（为什么 / 你以为 / 有人问 / 如果 / 你见过 / 到底有多）
- 必须含主线词的"情绪化版本"
- 画面是真实世界素材（影视/历史画面/实拍），不是游戏画面
- 字幕和口播一致

**don't**：
- ✗ 出现产品名 / 游戏 Logo
- ✗ 用纯陈述句（"驱逐舰很强大" 这种）
- ✗ 把主线词写成裸知识点（"以步制骑" 这种）
- ✗ 真人出镜（除非该号该季度专门策划）

### 段 2 · 铺垫A（4-14s，30-55 字）

**do**：
- 给出第一个证据/数据/案例来展开 Hook
- 句式："其次 / 而且 / 因为 / 不仅…还"
- 数字或专有名词出现 1-2 个（强化知识感）
- 画面继续真实世界

**don't**：
- ✗ 切到游戏画面（这是桥梁段才做的事）
- ✗ 出现产品名
- ✗ 空话铺垫（"这是个很复杂的问题"这种）

### 段 3 · 铺垫B（14-26s，35-60 字）

**do**：
- 给出第二个证据，并把情绪推到**全片峰值**
- 适合放最震撼的画面 + 最有杀伤力的金句
- 句式："更狠的是 / 更绝的是 / 更离谱的是 / 殊不知"

**don't**：
- ✗ 这里就开始切游戏（桥梁还没到）
- ✗ 平淡过渡（这是情绪峰值，不能拉低）

### 段 4 · 桥梁（26-33s，35-60 字）

**do**：
- 用最朴素的转折句把真世界一秒钟切到游戏
- 标准句式：
  - "然而在 [产品] 中…"
  - "而今在 [产品] 中…"
  - "但是在 [产品] 里…"
  - "所以当 [概念] 在 [产品] 中…"
- 画面在这一句话内完成从真实世界到游戏的切换
- BGM 在这里必须有节奏变化（峰值 → 游戏感）

**don't**：
- ✗ 用复杂的过渡句（"接下来让我们看看..."这种）
- ✗ 桥梁太长（35-60 字以内，1 句话最好）
- ✗ 没有节奏变化（BGM 不变 = 情绪断裂）

### 段 5 · 玩法A（33-41s，35-55 字）

**do**：
- **因果回答 Hook 的痛点**——必须能填入 H-1 模板的因果路径
- 只展示 1 个最紧扣 Hook 的卖点
- 卖点必须在产品方提供的卖点库里能找到出处
- 句式："你可以 / 你将亲身体验 / 让你 / 你不仅能"
- 画面 100% 游戏画面

**don't**：
- ✗ 罗列 3 个以上卖点
- ✗ 卖点和 Hook 没有因果关系
- ✗ 编游戏不存在的卖点
- ✗ 抽象描述（"游戏内容丰富"这种）

### 段 6 · 玩法B（41-47s，30-50 字）

**do**：
- **必须在玩法A 上做维度跳跃**（不是平行展示第二个卖点）
- 跳跃方向只能是以下 3 种之一：
  - 单兵 → 兵种群（"操控驱逐舰" → "指挥海陆空舰队"）
  - 个体能力 → 体系感（"一个英雄" → "整个文明体系"）
  - 单点体验 → 世界感（"一场战斗" → "横扫万国 / 称霸魔兽万人战场"）
- 句式："不仅…还 / 不止有…还 / 更绝的是 / 真正的 / 一体化 / 全方位"

**don't**：
- ✗ 玩法B 是玩法A 的"另一个并列卖点"——这是平庸罗列
- ✗ 没有"放大感 / 体系感 / 世界感"——读完没感觉游戏比预期大

### 段 7 · CTA（47-50s，15-25 字）

**do**：
- 含产品名
- 含 1 个动作动词（快来 / 立即下载 / 加入 / 一起来）
- 含 1 个核心体验描述（驰骋沙场 / 称霸战场 / 一统万国）
- 画面：游戏 Logo + 标语
- 字幕和口播一致

**don't**：
- ✗ 超过 25 字
- ✗ 出现"史诗级 / 顶级 / 巅峰"这种空话
- ✗ 字幕里出现版号 / 法律免责声明（这些在视频右上角处理，不进字幕）

## 8 张自审表（写完后必填）

### 表 1 · 人设外壳检查

| 项 | 通过条件 |
|---|---|
| Hook 不含产品名 | YES |
| 全片无真人出镜（除特别策划） | YES |
| 调性是"硬核知识/科普"，不是"喊话/煽情/媚俗" | YES |
| 整体语气：平稳叙述 + 略带调侃 | YES |

任一项 NO = FAIL

### 表 2 · 情绪燃料检查（最重要的新增表）

| 项 | 通过条件 |
|---|---|
| 显式选定 1 个主燃料（民族/智识/欲望落差） | 在 self_audit.emotional_fuel 填写 |
| Hook 自带情绪框架（不是裸知识点） | YES |
| 主线词是带燃料的版本（不是裸名词） | YES |
| 想象一个目标用户读完前 25 秒，他的第一反应是"爽了/想试/凭什么不是我们"，而不仅仅是"学到了" | YES |
| 整片至少 1 处"金句感"句子（不是单纯科普） | YES |

≥4 项 YES 才过

### 表 3 · 主线词与因果路径

| 项 | 通过条件 |
|---|---|
| 主线词在 Hook、铺垫A、铺垫B、桥梁、玩法A 共 5 段出现≥3 次（不必逐字） | YES |
| 玩法A 给出完整因果路径：现实痛点 → 卖点机制 → 玩家感知结果 | YES |
| 玩法A 卖点是产品真实存在的（不是 AI 编的） | YES |

任一项 NO = FAIL

### 表 4 · 玩法段维度跳跃

| 项 | 通过条件 |
|---|---|
| 玩法B 不是玩法A 的并列第二个卖点 | YES |
| 玩法B 体现以下之一：单兵→兵种群 / 个体→体系 / 单点→世界感 | YES |
| 玩法B 含至少 1 个"放大词"（不止/更绝的是/真正的/一体化/全方位/横扫/碾压） | YES |

任一项 NO = FAIL

### 表 5 · 字幕规则

| 项 | 通过条件 |
|---|---|
| 字幕只显示重点（不是口播全文照搬） | YES |
| 单行字幕 ≤ 14 字（参考头部样本中位数 13） | YES |
| 关键大字（如"中式文明"/"诸葛弩"）在画面里以大标题出现≥1 次 | YES |

任一项 NO = FAIL

### 表 6 · CTA 规则

| 项 | 通过条件 |
|---|---|
| CTA 含产品名 | YES |
| CTA 含动作动词 + 核心体验 | YES |
| CTA ≤ 25 字 | YES |
| CTA 不出现"史诗/顶级/巅峰"等空话 | YES |

任一项 NO = FAIL

### 表 7 · 换名检验（最终防线）

把脚本里所有产品名替换成竞品名（万国 → 列王的纷争 / 战火 → 现代战争 / 万龙 → 龙腾世界），整段读下来：

| 通过条件 |
|---|
| 整段读下来明显**不顺畅 / 不成立 / 卖点漏洞百出** = PASS |
| 整段读下来仍然顺畅 = **FAIL，必须重写玩法段** |

### 表 8 · Hook 差异化检查（防 few-shot 过拟合）

把你写的 Hook 与 04-few-shot 8 条样本 Hook 逐一对比：

| 通过条件 |
|---|
| Hook 与每条样本 Hook 的字面重合度均 ≤ 40%（只能共享钩子词和框架词） = PASS |
| Hook 与任一样本 Hook 字面重合 > 40%（只换了对象名）= **FAIL，必须重写 Hook** |

**重合度计算**：把两个 Hook 拆成字串，计算重叠字符比例。例：
- 样本"古代士兵明明每天都要吃饭，为何一打仗就缺粮"（23 字）
- 你写的"古代士兵明明每天都要吃肉，为何一打仗就缺肉"（23 字）→ 重合 19 字 = 83% → **FAIL**
- 你写的"古代攻城那么难，为何中国人独有的水攻能一夜破城"（23 字）→ 重合 3 字（"古代 / 为何"）= 13% → **PASS**

## 输出格式

模型必须输出**两块内容**，用 `===SCRIPT===` 和 `===AUDIT===` 分隔。

### 输出块 1 · 可读脚本（Markdown）

```markdown
# [脚本标题]

**游戏**：[产品名] | **时长**：[Xs] | **目标卖点**：[卖点] | **主情绪燃料**：[民族/智识/欲望落差]

## Hook
[Hook 口播文案，20-35 字]

## 主线词
具体层：[XX]
抽象层：[XX]

## 分镜表

| 段位 | 时间 | 画面 | 字幕 | 口播 | BGM |
|---|---|---|---|---|---|
| Hook | 0-4s | [画面描述] | [字幕] | [口播] | [BGM] |
| 铺垫A | 4-14s | … | … | … | … |
| 铺垫B | 14-26s | … | … | … | … |
| 桥梁 | 26-33s | … | … | … | … |
| 玩法A | 33-41s | … | … | … | … |
| 玩法B | 41-47s | … | … | … | … |
| CTA | 47-50s | … | … | … | … |

## CTA 文案
[完整 CTA 文案，15-25 字]
```

```json
{
  "schema_version": "v0.9",
  "product": "万国觉醒",
  "duration_seconds": 50,
  "main_keyword": {

  "causal_path": {

  "feature_authenticity": {

  },
  "self_audit": {
    "persona_check": "PASS|FAIL",
    "emotional_fuel_check": "PASS|FAIL（≥4 项YES）",
    "main_keyword_check": "PASS|FAIL",
```
