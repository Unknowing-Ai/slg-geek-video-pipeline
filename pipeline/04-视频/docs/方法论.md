## 二、5 条视频生成铁律（PoC 实测推导）

### 铁律 1：动作连续多视角（最重要）

**错误理解**：matched cut = 同主题不同瞬间（如运粮全景 → 冰冻骑兵特写 → 粮草地图）
**正确理解**：matched cut = 同一持续动作的不同机位（如骑兵冲锋正面 → 战马蹄部俯视 → 骑兵列队侧面）

**判断标准**：能不能用一句「主体 + 持续动词」描述整组镜头？
- ✓ 「骑兵正在持续冲锋」 → 任何机位都属于同一动作组
- ❌ 「关于古代行军的杂集」 → 不是同一动作，是同一主题

**落地**：
- storyboard 每个 sequence_group 必须用「主体+动词」命名（如 `cavalry_charging`、`wheel_stuck_spinning`），禁用主题词
- 同组镜头的 visual_description 共享 70% 关键词，只在【机位/景别/动作进行阶段】变化
- 首帧图必须是「动作进行中」姿态（蹄子半抬、士兵正用力推、车轮带速度感），不能是静态站位

**实测教训**：PoC v1 「汉军行军」组用 3 个独立镜头（运粮 / 推车 / 粮草），看着仍有断裂感；PoC v2 用「同一支汉军在持续行军」3 个机位（正面远 / 蹄部特 / 侧面中），连贯感质变

### 铁律 2：主体内部动态（克服模型惰性）

**问题**：模型对「轮子陷泥」「将军凝视」等被动描述的默认理解 = 「静态图 + 镜头推近」，缺动态感

**正确做法**：video prompt 必须显式描述主体的动作循环

- ❌ 静态："a war horse standing in deep snow, camera slowly zooms in"
- ✓ 动态循环："a war horse's hoof rhythmically lifts and stomps into the snow with multiple full strike cycles, snow particles spraying outward, leg muscles flexing"

**关键短语库**（写进 prompt）：
- 动作循环：`rhythmically [verb], multiple full cycles`
- 部件运动：`legs cycling, hooves stomping, wheels spinning, fabric flowing, mane swaying`
- 生理动态：`breath vapor visible, eyes blinking, jaw moving, fingers gripping`
- 环境响应：`particles flying outward, dust kicked up, snow splashing, water rippling`

**NEG 强约束**：`static image, no movement, freeze frame, slow motion pause`

### 铁律 3：文化/视觉统一（跨产品参数化）

**问题**：模型在「古代战争」「中世纪」「未来战场」等 prompt 下会默认混入欧美面孔/现代装备/错误文化元素

**正确做法**：每个产品在 `product_config.yaml` 中定义两个锚定字段：

```yaml
part1_anchor: "<Part 1 文化/服装/时代锚定，由宿主提供>"
part2_anchor: "<Part 2 游戏画风锚定>"
part1_neg_extra: "<Part 1 排除元素>"
part2_neg_extra: "<Part 2 排除元素>"
```

**示例（不同产品）**：

| 产品 | part1_anchor | part1_neg_extra |
|---|---|---|
| ROK 万国觉醒 (古代) | "Ancient civilization era with culturally consistent characters and period-accurate costumes; Han Dynasty for Chinese segments, Roman for Roman" | "non-asian people in Chinese segments, european faces in Han Dynasty" |
| WGAME 战火勋章 (现代军事) | "Modern military equipment and uniforms (NATO standard), realistic combat photography" | "medieval armor, ancient weapons, swords, fantasy elements" |
| Samo 万龙觉醒 (西方奇幻) | "Western medieval fantasy with dragons knights mages, Tolkien-inspired costumes" | "modern military, real-world country flags" |
| 日韩二次元题材 | "Anime-styled characters with Japanese/Korean aesthetic, vibrant cel-shading" | "photorealistic faces, western art style" |
| 科幻题材 | "Sci-fi future setting with futuristic suits and high-tech equipment" | "medieval or historical elements, fantasy magic" |

### 铁律 4：禁英文 / Logo / Download（按投放区域）

**问题**：中国境内投放素材出现英文 Download 按钮、Logo 拉丁字符乱码

**正确做法**：
1. AI 视频/图**只画画面，不画任何文字**
2. NEG 强约束：`latin letters, english characters, download button, click here`（中国投放）
3. 所有中文字幕、Logo、按钮**由剪辑师后期合成**，AI 不负责

**按投放地区配置 forbidden_language_neg**：

| 投放区域 | forbidden_language_neg |
|---|---|
| 中国境内 | `latin letters, english characters, download button, click here` |
| 海外英文区 | `chinese characters, japanese characters, korean characters` |
| 阿语区 | `latin letters, english characters, chinese characters` |

### 铁律 5：训练资产留口（宿主资产升级）

**问题**：宿主只提供静态立绘，Part 2 sequence_group 用同一立绘做多镜 reference → 模型只能复制立绘姿势，导致「同立绘 + 镜头推拉」的伪连贯

**正确做法**：要求宿主额外提供「动作姿态资产」(action_keyframes)

每个 P0 角色应该有 3-5 张不同动作/姿态/视角：
- 立绘（正面静立）
- 冲锋动作（侧面动态）
- 释放技能（特效中）
- 胜利姿态（背面）
- 兵种作战（俯视/特写）

详见 [`prompts/asset_request_template_v2.md`](../prompts/asset_request_template_v2.md)

## 三、跨产品复用流程

### Step 0 · 跟宿主要资产

按 [`asset_request_template_v2.md`](../prompts/asset_request_template_v2.md) 给宿主发请求，6 大类资产：
1. 主要英雄立绘（P0）
2. **主要英雄动作姿态 3-5 张 ⭐NEW**（P0）
3. 主要兵种/怪物（P1）
4. 代表性建筑/环境（P2）
5. LOGO/IP 标识（P2，后期合成）
6. **文化/视觉锚定文字描述 ⭐NEW**（P0）

### Step 1 · 整理配置

- `manifest.json`：资产清单（立绘 + 动作 keyframes）
- `product_config.yaml`：从 [`product_config_template.yaml`](../prompts/product_config_template.yaml) 复制，宿主填 7 个字段
- `shot_to_refs_mapping.json`：分镜→资产映射（含动作 keyframe 映射）

### Step 2 · 跑 4 步 pipeline

```bash
# A. 压缩首帧图 + 上传
python scripts/01_upload_frames.py <storyboard.json> <img_dir> <out_dir>

# B. 提交视频生成（产品参数化）
python scripts/02_gen_video.py <storyboard.json> <out_dir>/frame_assets.json <product_config.yaml> <out_dir>

# C. poll + 下载 + 失败 fallback
python scripts/03_poll_download.py <out_dir>/video_tasks.json <storyboard.json> <product_config.yaml> <out_dir>

# D. 按 ASL 裁切 + Hard_Cut 拼接 60s
python scripts/04_cut_concat.py <storyboard.json> <out_dir> [desktop_dir]
```

## 五、常见问题排查

| 现象 | 根因 | 修复 |
|---|---|---|
| 单镜「静态图 + 镜头推近」 | 铁律 2 主体内部动态没落地 | video prompt 加 `rhythmically [verb], multiple full cycles` |
| 同 sequence_group 看着碎 | 铁律 1 没用「同动作多视角」，是「同主题多瞬间」 | storyboard sequence_group 改名「主体+动词」，同组共享 70% 关键词 |
| 古代/历史画面有欧美面孔 | 铁律 3 part1_anchor 没配 | product_config 加 `part1_anchor` 强文化约束 + NEG |
| 视频里出现 Download 按钮 | 铁律 4 forbidden_language_neg 没配 | product_config 加投放地区禁词 |
| Part 2 角色动作只是「立绘 + 镜头推」 | 铁律 5 没要动作 keyframes | 找宿主补「动作姿态资产」3-5 张/角色 |
| atlas-skillhub HTTP 413 | PNG > 1MB | 先 ffmpeg 压成 1920w JPG (~500KB) |
| seedance 安全拦截（多国家旗帜） | Google 安全机制 | 03 脚本自动 fallback 到 kling |
| 并发 4 频繁失败 | atlas-skillhub gateway 容量 | 降到并发 2，失败串行重试 |
