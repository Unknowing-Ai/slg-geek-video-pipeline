# Step 4 字幕训练方法论 v1.0

> 极客号字幕从 codex 抽取到工程渲染的完整方法论。状态：v1.0（PoC 5 条样本验证 + 端到端 v3 视频实测通过）

## 一、问题定义

**字幕的核心职责**：在不打断画面/口播节奏的前提下，为口播信息提供视觉冗余 → 提升信息密度 + 移动端无声播放友好。

**为什么字幕做不好整个视频废一半**：
1. 错别字/品牌词错（"万国觉醒"识别成"万国决心"）→ 直接砍掉转化
2. 字幕与口播错位 → 用户认知撕裂
3. 字太小看不清 → 移动端无效曝光
4. 风格不对（普通 fade_in）→ 极客号"硬核买量"质感全无

## 二、3 大铁律（用户原始诉求）

| 原则 | 失败表现 | 落地约束 |
|---|---|---|
| **跟口播高度匹配** | 字幕和口播说的内容不一致 | 强制用原始 script.yaml 文本，禁用 ASR |
| **切换无缝丝滑** | 字幕慢半拍 / 提前出 / 卡两秒不切 | 段位时间映射 + 段内按字数比例分时 |
| **美观大方** | 字体山寨 / 字小看不清 / 描边糊 | 思源黑体 Bold + 96px + 5px 描边 + Spacing 4 |

## 三、关键工程决策

### 决策 1：用脚本不用 ASR

**问题**：直觉以为"先 ASR 出文本再做字幕"是标准做法，PoC v1 跑完发现 Whisper base 在 TTS 音质 + 军事品牌专业词上误识率高。

**实测错误样本**：
| 原文 | Whisper base 识别 | 影响 |
|---|---|---|
| 万国觉醒 | 万国决心 | **品牌词错，直接砍转化** |
| 匈奴 | 凶奴 | 历史错乱 |
| 冻死 | 动死 | 错别字 |
| 中式弓骑兵 | 中市攻击兵 | 卖点错乱 |
| 起兵 | 齐兵 | 错别字 |
| 降维优势 | 降位优势 | 卖点错乱 |
| 名将 | 明将 | 错别字 |

**结论**：直接用原始 script.yaml 文本，时间戳按段位映射 + 段内字数比例分配（**不需要 forced alignment**，因为段位级精度对齐就够了，秒级精度足以覆盖人眼感知阈值）。

### 决策 2：段位时间映射用"实际配音"不用"目标 duration"

```python
# 错误做法（脚本里的 target）
SEG_TIMES = {
    "Hook_Counter_Intuitive": (0, 5),     # 脚本写 5s
    "Core_Knowledge": (5, 27),            # 脚本写 22s
    ...
}

# 正确做法（实际配音音轨）
# raw 配音 46.15s，mixed 39.21s（含 BGM）
# 各段 raw duration × (39.21 / 46.15)
SEG_TIMES = {
    "Hook_Counter_Intuitive": (0, 4.152 * 0.85),
    "Core_Knowledge": (4.152*0.85, (4.152+18.36)*0.85),
    ...
}
```

**Why**：脚本里写"Hook 5s"是目标值，实际 TTS 出来可能 4.15s 或 5.3s。字幕必须贴**实际**音轨，否则前一段拖到后一段口播开始时还在显示。

### 决策 3：段内按字数比例分时 ≠ 平均分时

```python
# 错误：N 句平均分
per = (seg_end - seg_start) / N

# 正确：按字数权重
total_chars = sum(len(s) for s in sentences)
for sentence in sentences:
    dur = (seg_end - seg_start) * len(sentence) / total_chars
```

**Why**：句子长度差异大时（"零损耗。" vs "30多位真实历史名将随你调度，"），平均分时会让短句停留过久、长句一闪而过，与口播节奏脱节。

### 决策 4：CTA 段位有独立样式集

PoC 5/5 一致观察：尾段 CTA 的字幕样式与正文段位**明显不同**：

| 维度 | Normal 段位 | CTA 段位 |
|---|---|---|
| 字号 | 96px (6% 屏高) | 144px (13% 屏高) |
| 字体 weight | Bold | Heavy |
| 字体 italic | 否 | **是** |
| 描边宽度 | 5px | 7px |
| 动画 | 无 (hard_cut) | popup (fad 150ms) |
| 高亮色 | 金黄 #FFD700 | 警示红 #FF0000 |
| 高亮频率 | 中（关键节点） | 高（密集） |

**Why**：CTA 是转化诉求最强的段位，字幕样式独立强化"立即行动"压感。

### 决策 5：字体必须装思源黑体

PoC v1 用 `WenQuanYi Zen Hei`（系统自带）渲染，用户反馈"字体不清晰"。换成 `Source Han Sans SC Bold/Heavy` 后立即接受。

**Why**：思源黑体是字形覆盖度 + 字重梯度 + 中文渲染优化最好的开源字体，与极客号风格（思源黑体/阿里巴巴普惠体/微软雅黑）100% 同档。

### 决策 6：字符间距 Spacing 4-6 必须加

默认 Spacing 0 会让"中式弓骑兵"这种长词字符黏连，移动端看不清。

**Why**：96px Bold 字体在 1080p 上字符宽度约 100px，4-6px spacing 让字符呼吸感增强 ~5%，可读性显著提升。

### 决策 7：字幕末尾标点必须剥离

极客字幕实测：每条字幕末尾无任何标点符号。脚本里的"，""。""？""！"用于**切句位置定位**后必须剥离，否则视觉上极度冗余。

```python
# 切句后处理
line = re.sub(r'[，。？！,.?!、；;：:]+$', '', line)
```

**保留例外**：句中并列顿号"、"必须保留 — 如"冻死、饿死、迷路掉队"去掉就读不通。规则是**仅剥离字幕末尾**的标点。

**Why**：标点对口播是必要的（决定停顿），对字幕是噪音（视觉冗余 + 占字符位）。极客号字幕从不显示句末标点，是节奏感的视觉表达。

## 四、跨产品适配机制

字幕**样式 codex 全产品通用**（位置/字体/颜色/切换/段位级差异）。需按产品定制的只有**高亮词典**：

```yaml
# product_config.yaml 新增 2 个字段
highlight_words_normal:
  - "{game_name}"              # 必填，品牌词
  - <核心角色 5-10 个>          # ROK: 霍去病/卫青/吕布；WGame: 阿帕奇/狙击手...
  - <数字 pattern: \d+万 \d+%>
  - <卖点关键词>

highlight_words_cta:
  - "{game_name}"
  - "下载" / "现在" / "立即" / "免费"
  - <CTA 文案核心词>
```

**Why**：字幕样式是"形"，高亮词是"神"。形跨产品一致，神按产品换。

## 五、Pipeline 工程实现

```
[输入]
  ├── video.mp4 (来自 step3，已带配音)
  ├── script.yaml (来自 step1，原始口播文本)
  └── product_config.yaml (含 highlight 词典)
        │
        ▼
[Phase 1] 读脚本 + 计算段位实际时间
  - 按 raw / mixed 比例 scale 段位时间
  - 输出 SEG_TIMES dict
        │
        ▼
[Phase 2] 段内切句
  - 按中文标点切（，。？！）
  - 长句硬切到 ≤22 字/条
  - 按字数比例分配时间
        │
        ▼
[Phase 3] 高亮规则应用
  - Normal 段位：黄高亮（关键词词典 1）
  - CTA 段位：红高亮（关键词词典 2 + popup）
  - 按词长降序匹配避免子串重复（"现在下载" 优先于 "现在"）
        │
        ▼
[Phase 4] 生成 .ass + ffmpeg burn
  - Style 表 2 套（Normal + CTA）
  - Dialogue 行按 (start, end, style, text) 输出
  - ffmpeg -vf subtitles=xxx.ass 烧到视频
        │
        ▼
[输出] {video}_字幕版.mp4
```

总耗时：~30s（不含视频本身，5 段 26 字幕的烧字幕）。

## 七、局限 + 后续迭代

### v1.0 局限
1. **段内分时是字数比例近似**，不是真正 forced alignment（误差 ±200ms 内）— 90% 场景够用，未来可加 whisperx 强制对齐做精度补充
2. **段位边界依赖人工标定**（SEG_TIMES dict）— 需要从 step1 配音输出自动 export 标准段位时间表
3. **高亮词典手动维护** — 每个新产品要手填 highlight_words，未来可由 LLM 从 script 自动提取
4. **画幅适配缺位**（step3.5 待开发） — 当前只输出 1920×1080 16:9，字幕在 9:16 / 1:1 上未渲染

### v1.x 迭代方向
- [ ] step1 配音输出标准 `seg_times.yaml`，step4 自动读取（解耦段位时间硬编码）
- [ ] LLM 从 script 自动提取关键词高亮候选（人审后入 product_config）
- [ ] step3.5 画幅适配集成（详见 [04.5-画幅/SKILL.md]）
- [ ] 字幕样式 codex v2.0（扩 25-30 条样本验证 CTA 规则跨产品稳定性）
- [ ] 支持"画面中央飞字大字"模块（PoC 观察到，schema 已留口）
